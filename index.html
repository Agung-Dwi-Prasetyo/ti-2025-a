üìö Apa itu GitHub?

GitHub adalah platform hosting kode berbasis cloud yang memungkinkan developer untuk menyimpan, mengelola, dan berkolaborasi dalam proyek pemrograman. GitHub menggunakan sistem version control bernama Git untuk melacak perubahan kode.

GitHub Desktop adalah aplikasi yang memudahkan kamu untuk berinteraksi dengan GitHub tanpa perlu menggunakan command line. Sangat cocok untuk pemula!

üíª Instalasi GitHub Desktop

  1. Download GitHub Desktop

    Kunjungi website resmi dan download aplikasi sesuai sistem operasi kamu

    Download GitHub Desktop
  2. Install Aplikasi

    Buka file installer yang telah didownload dan ikuti instruksi instalasi

  3. Sign In

    Buka GitHub Desktop dan sign in dengan akun GitHub kamu. Jika belum punya akun, daftar terlebih dahulu di github.com

  4. Konfigurasi Git

    Masukkan nama dan email yang akan digunakan untuk commit kamu

üóÇÔ∏è Membuat Repository Baru

Opsi 1: Membuat Repository Lokal

  1. Klik "Create New Repository"

    Pada halaman utama GitHub Desktop, pilih "File" ‚Üí "New Repository"

  2. Isi Detail Repository

    - Name: Nama project kamu
    - Description: Deskripsi singkat project
    - Local Path: Lokasi folder di komputer kamu
    - Initialize with README: Centang untuk membuat file README.md

  3. Klik "Create Repository"

    Repository lokal kamu sudah dibuat!

Opsi 2: Clone Repository yang Sudah Ada

  1. Klik "Clone Repository"

    Pilih "File" ‚Üí "Clone Repository"

  2. Pilih Repository

    Pilih dari daftar repository kamu atau masukkan URL repository

  3. Pilih Lokasi

    Tentukan di mana kamu ingin menyimpan repository di komputer kamu

üîÑ Workflow Dasar GitHub

  1. Membuat/Mengubah File

    Buat atau edit file dalam folder repository kamu menggunakan text editor atau IDE favorit

  2. Review Changes

    GitHub Desktop akan otomatis mendeteksi perubahan. Cek tab "Changes" untuk melihat file yang berubah

  3. Commit Changes

    - Tulis commit message yang jelas dan deskriptif di bagian bawah kiri
    - Commit message menjelaskan apa yang kamu ubah
    - Klik tombol "Commit to main"

    üí° Tips Commit Message yang Baik:
    ‚úÖ "Menambahkan fitur login"
    ‚úÖ "Memperbaiki bug pada halaman dashboard"
    ‚ùå "update"
    ‚ùå "fix"
  4. Push ke GitHub

    Setelah commit, klik tombol "Push origin" untuk mengirim perubahan ke GitHub

  5. Pull dari GitHub

    Jika ada perubahan dari orang lain, klik "Fetch origin" lalu "Pull origin" untuk mendapatkan update terbaru

üåø Bekerja dengan Branch

Branch memungkinkan kamu untuk bekerja pada fitur baru tanpa mengubah kode utama (main branch). Ini sangat berguna untuk eksperimen atau development fitur.

  1. Membuat Branch Baru

    Klik "Current Branch" ‚Üí "New Branch" ‚Üí Beri nama (contoh: "fitur-login") ‚Üí "Create Branch"

  2. Bekerja di Branch

    Lakukan perubahan, commit seperti biasa. Perubahan hanya terjadi di branch ini

  3. Pindah Branch

    Klik "Current Branch" dan pilih branch yang ingin kamu gunakan

  4. Merge Branch

    Setelah fitur selesai, klik "Current Branch" ‚Üí "Choose a branch to merge into main" ‚Üí Pilih branch yang ingin digabungkan

üë• Kolaborasi dengan Tim

Menambahkan Collaborator

  1. Buka Repository di GitHub.com

    Klik "Repository" ‚Üí "View on GitHub" di GitHub Desktop

  2. Tambahkan Collaborator

    Settings ‚Üí Collaborators ‚Üí Add people ‚Üí Masukkan username atau email teman kamu

Pull Request

Pull request adalah cara untuk mengusulkan perubahan sebelum digabungkan ke branch utama

  1. Push Branch

    Push branch dengan fitur baru ke GitHub

  2. Create Pull Request

    Klik "Branch" ‚Üí "Create Pull Request" atau buat di GitHub.com

  3. Review dan Merge

    Tim bisa mereview kode, memberikan komentar, lalu merge jika sudah oke

üí° Tips dan Best Practices

üéØ Commit

  • Commit sesering mungkin dengan perubahan yang kecil dan spesifik
  • Tulis commit message yang jelas dan informatif
  • Hindari commit file yang tidak perlu (node_modules, .env, dll)

üåø Branch

  • Gunakan branch untuk setiap fitur atau bug fix baru
  • Beri nama branch yang deskriptif (contoh: fitur-keranjang, fix-bug-login)
  • Hapus branch yang sudah tidak digunakan

üë• Kolaborasi

  • Selalu pull sebelum mulai bekerja untuk mendapat update terbaru
  • Komunikasikan dengan tim sebelum merge ke branch utama
  • Gunakan pull request untuk code review
  • Selesaikan conflict dengan hati-hati

üìù File .gitignore

  • Buat file .gitignore untuk mengecualikan file yang tidak perlu di-track
  • Contoh file yang diabaikan: node_modules/, .env, .DS_Store
  • Template gitignore tersedia di github.com/github/gitignore

üîß Troubleshooting

Masalah Umum dan Solusi

‚ö†Ô∏è Merge Conflict

Terjadi ketika dua orang mengubah bagian yang sama dari file.

Solusi: Buka file yang conflict, pilih perubahan mana yang mau dipertahankan, lalu commit hasil resolve conflict.

‚ö†Ô∏è Push Ditolak

Terjadi ketika ada perubahan di remote yang belum ada di lokal kamu.

Solusi: Pull terlebih dahulu, resolve conflict jika ada, lalu push lagi.

‚ö†Ô∏è Commit ke Branch yang Salah

Tidak sengaja commit ke branch yang salah.

Solusi: Di GitHub Desktop, klik "History" ‚Üí klik kanan pada commit ‚Üí "Revert This Commit" (jika belum push) atau buat branch baru dari commit tersebut.
